# AI 및 데이터 설계

## 1. AI가 필요한 이유

팝업스토어 운영자는 오픈 전에 방문자 수, 인기 상품, 혼잡 시간대를 예측해야 한다. 하지만 사람의 감이나 SNS 반응만으로는 정확한 판단이 어렵다.

AI는 사전 참여 데이터를 바탕으로 여러 변수를 함께 고려하여 운영자에게 의사결정에 필요한 예측 결과를 제공할 수 있다.

## 2. 수집 데이터

| 데이터 | 예시 | 활용 목적 |
| --- | --- | --- |
| 방문 의향 | 방문 예정, 관심 있음, 미정 | 예상 방문자 수 계산 |
| 방문 희망 시간 | 금요일 18시, 토요일 14시 | 시간대별 혼잡도 예측 |
| 관심 상품 | 티셔츠, 키링, 한정 굿즈 | 상품별 수요 예측 |
| 예상 구매 금액 | 1만원, 3만원, 5만원 이상 | 예상 매출 추정 |
| 사용자 지역 | 성북구, 종로구, 마포구 | 지역 기반 수요 분석 |
| 동반 인원 | 혼자, 2명, 3명 이상 | 실제 방문 규모 보정 |
| 리워드 반응 | 쿠폰 클릭, 예약권 사용 | 방문 전환 가능성 예측 |
| 실제 방문 기록 | QR 체크인 | 예측 모델 개선 |
| 실제 구매 기록 | 구매 금액, 구매 상품 | 상품 수요 예측 개선 |

## 3. 예측 항목

### 3.1 방문 수요 예측

사전 설문 참여자 수, 방문 의향 강도, 예약 등록 수, 리워드 사용 의향을 바탕으로 실제 방문 가능 인원을 예측한다.

예시 결과:

- 예상 총 방문자: 1,200명
- 높은 확률 방문자: 620명
- 중간 확률 방문자: 410명
- 낮은 확률 방문자: 170명

### 3.2 상품 수요 예측

관심 상품 선택과 예상 구매 금액을 바탕으로 상품별 예상 판매량을 추정한다.

예시 결과:

- 한정 키링: 430개 권장
- 티셔츠: 180장 권장
- 스티커팩: 300개 권장
- 포스터: 120장 권장

### 3.3 혼잡도 예측

방문 희망 시간과 예약 데이터를 이용해 특정 시간대에 방문객이 몰릴 가능성을 예측한다.

예시 결과:

- 토요일 13시-15시: 매우 혼잡
- 토요일 17시-19시: 혼잡
- 일요일 11시-13시: 보통
- 평일 15시-17시: 여유

## 4. AI 모델 접근 방법

초기 단계에서는 복잡한 딥러닝보다 설명 가능한 머신러닝 또는 점수 기반 모델이 적합하다.

가능한 접근:

- 회귀 분석: 예상 방문자 수, 예상 매출 예측
- 분류 모델: 실제 방문 가능성 높음/중간/낮음 분류
- 군집 분석: 비슷한 관심사를 가진 소비자 그룹 분류
- 추천 알고리즘: 사용자 관심사에 맞는 팝업 또는 쿠폰 추천
- 시계열 분석: 시간대별 방문 흐름 예측

## 5. 예측 점수 예시

실제 구현 초기에 사용할 수 있는 간단한 점수식은 다음과 같다.

```text
방문 가능성 점수 =
  방문 의향 점수
  + 예약 완료 점수
  + 관심 상품 선택 점수
  + 쿠폰 저장 점수
  + 과거 방문 이력 점수
```

점수가 높은 사용자는 실제 방문 가능성이 높은 그룹으로 분류한다. 이후 실제 QR 체크인 데이터를 쌓으면 모델을 점차 개선할 수 있다.

## 6. 데이터 윤리와 개인정보 보호

개인정보를 다루는 시스템이므로 다음 원칙이 필요하다.

- 필요한 데이터만 최소한으로 수집한다.
- 개인을 식별할 수 있는 정보는 암호화하거나 익명화한다.
- 사용자는 데이터 수집 목적을 명확히 확인할 수 있어야 한다.
- 예측 결과는 개인을 차별하는 방식으로 사용하지 않는다.
- 지역, 연령, 성별 등의 정보는 통계적 분석 목적으로만 사용한다.

## 7. 기술적 실현 가능성

이 시스템은 현재 널리 쓰이는 웹 기술, 클라우드 데이터베이스, 데이터 분석 도구, 머신러닝 기법으로 구현 가능하다. 특히 초기 버전은 실제 AI 모델 없이도 사전 설문 데이터와 규칙 기반 점수화를 통해 개념 검증이 가능하다.
