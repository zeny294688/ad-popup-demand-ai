# 문제 정의

## 1. 착안 배경

팝업스토어는 짧은 기간 동안 특정 장소에서 브랜드, 상품, 지역 콘텐츠를 체험하게 하는 오프라인 마케팅 방식이다. 최근에는 대형 브랜드뿐 아니라 지역 소상공인, 청년 창업자, 로컬 크리에이터도 팝업스토어를 활용한다.

하지만 팝업스토어는 운영 기간이 짧고 준비 비용이 집중되기 때문에, 수요 예측에 실패하면 손실이 빠르게 발생한다.

## 2. 현실의 불편

운영자 입장에서 가장 어려운 점은 “얼마나 많은 사람이 올지”와 “무엇을 얼마나 준비해야 하는지”를 알기 어렵다는 것이다.

대표적인 문제는 다음과 같다.

- 방문객이 예상보다 적으면 공간 대여료, 인건비, 재고 비용이 낭비된다.
- 방문객이 예상보다 많으면 대기 줄이 길어지고 상품 품절이 발생한다.
- 인기 상품을 잘못 예측하면 팔릴 상품은 부족하고 안 팔리는 상품은 남는다.
- 소비자는 긴 대기 시간 때문에 방문을 포기하거나 부정적인 경험을 하게 된다.
- 지역 상권은 방문객을 유치할 기회를 놓치고, 소규모 브랜드는 다음 시도를 하기 어려워진다.

## 3. 기존 방식의 한계

현재 많은 팝업스토어는 SNS 반응, 브랜드 인지도, 과거 경험을 바탕으로 수요를 추정한다. 이 방식은 빠르지만 객관적인 데이터가 부족하다.

특히 지역 소상공인이나 신규 브랜드는 기존 고객 데이터가 충분하지 않아 더 불리하다. 광고를 많이 집행하기도 어렵고, 데이터를 분석할 인력도 부족하다.

## 4. 해결해야 할 핵심 문제

이 프로젝트가 해결하려는 문제는 다음 한 문장으로 정리할 수 있다.

> 지역 팝업스토어 운영자가 오픈 전에 실제 방문 가능성과 상품 수요를 데이터 기반으로 예측할 수 없다는 문제

## 5. 프로젝트 목표

사전 참여 데이터를 수집하고 AI로 분석하여 팝업스토어 운영자가 다음 의사결정을 더 정확하게 할 수 있도록 돕는다.

- 어느 지역에서 팝업을 열 것인가
- 몇 명 정도 방문할 것인가
- 어떤 상품을 얼마나 준비할 것인가
- 어느 시간대에 방문객이 몰릴 것인가
- 리워드를 어떻게 제공해야 실제 방문으로 이어질 것인가

## 6. 성공 기준

개념 설계 프로젝트이므로 실제 구현 성능이 아니라 시스템이 문제 해결에 기여할 수 있는지를 기준으로 판단한다.

- 사전 데이터 수집 방식이 현실적으로 가능한가
- AI 분석 결과가 운영 의사결정에 직접 연결되는가
- 소비자가 참여할 동기가 충분한가
- 지역사회 문제 해결과 연결되는가
- 현재 기술로 가까운 미래에 구현 가능한가
